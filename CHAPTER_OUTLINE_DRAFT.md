# CMS教程章节编排草案（v1）

> 教程目标：在 AI 辅助下，从零完成一个“小型但完整可用”的 CMS。  
> 技术原则：小型、现代、够用、可部署。

## 第0章：学习方式与项目边界
**预计时间：0.5 小时**

- 本教程教什么 / 不教什么（不覆盖高并发、分布式等）
- 最终成品展示（前台 + 后台 + 数据库）
- 开发环境、工具链、AI协作方式总览
- 学习路径与每章产出说明

**章末练习（示例）**
- 说出本教程的 3 个“明确不覆盖”范围
- 画出你理解的最终系统模块图（前台/后台/数据库）

## 第1章：基础技术快速复习（HTML/CSS/JS/TS/SQL）
**预计时间：2 小时**

- HTML 语义结构与表单
- CSS 基本布局与后台页面常见样式
- JS 事件驱动与 DOM 操作
- TS 的类型约束价值（相对 JS）
- SQL 的 CRUD 与基础约束

> 本章强调“够用复习”，不深挖。

**章末练习**
- 手写一个登录页（HTML+CSS）
- 用 JS/TS 完成一个表单校验
- 写出文章表的基础 CRUD SQL

## 第2章：CMS需求分析与信息架构设计
**预计时间：1.5 小时**

- 最小可用 CMS 功能清单（MVP）
- 用户角色：管理员/编辑/访客
- 内容模型设计：文章、分类、标签、用户
- 前后台边界与路由草图
- 数据流与状态流说明

**章末练习**
- 输出你的 MVP 功能列表（不超过 12 条）
- 画出实体关系草图（ER 简图）

## 第3章：技术选型与项目初始化
**预计时间：1.5 小时**

- 现代且轻量的技术栈选择理由
- 前端、后端、数据库、ORM、认证方案确定
- 项目目录结构设计
- 本地开发环境搭建
- AI 提示词模板：初始化项目脚手架

**章末练习**
- 独立初始化项目并跑通 hello-page
- 解释为何该栈“适合小型 CMS”

## 第4章：数据库设计与迁移管理
**预计时间：2 小时**

- 表结构设计（users/posts/categories/tags/relations）
- 索引、唯一约束、外键策略
- 迁移（migration）流程
- 种子数据（seed）与开发样本数据

**章末练习**
- 完成建表迁移脚本
- 插入 10 条测试文章并完成查询

## 第5章：后端核心API开发（内容管理）
**预计时间：3 小时**

- RESTful API 设计规范
- 文章的增删改查
- 分类/标签管理
- 分页、筛选、排序
- 输入校验与统一错误响应

**章末练习**
- 完成“文章列表 + 条件筛选”接口
- 设计并实现统一错误码表

## 第6章：认证与后台权限控制
**预计时间：2 小时**

- 登录、会话/JWT 方案
- 角色权限（RBAC）基础
- 后台接口鉴权中间件
- 最小安全实践（密码哈希、基础防护）

**章末练习**
- 实现管理员登录与受保护接口
- 完成“未登录/权限不足”测试用例

## 第7章：后台管理界面开发
**预计时间：3 小时**

- 后台布局（侧栏、顶栏、内容区）
- 文章管理页（列表/新建/编辑）
- 分类标签管理页
- 表单与富文本输入（轻量实现）

**章末练习**
- 完成文章发布工作流（草稿→发布）
- 添加一个“快速搜索文章”功能

## 第8章：前台展示系统开发
**预计时间：2.5 小时**

- 首页、文章详情页、分类页
- SEO 基础（title/description/slug）
- Markdown/富文本渲染策略
- 前台与后台数据联动

**章末练习**
- 完成可访问的前台文章页
- 支持按分类查看文章列表

## 第9章：AI辅助开发实战方法
**预计时间：1.5 小时**

- 如何给 AI 提需求（上下文、约束、验收标准）
- 如何让 AI 生成更稳定代码
- AI 参与重构、排错、测试、文档
- 常见误区：盲信输出、忽视验证

**章末练习**
- 写 3 个高质量提示词（建模/API/调试）
- 对 AI 生成代码做一次人工审查清单

## 第10章：测试、部署与上线运维（小型服务器）
**预计时间：2.5 小时**

- 基础测试策略（接口测试/关键流程测试）
- 生产构建与环境变量管理
- Linux 单机部署（进程守护、反向代理、HTTPS）
- 日志、备份与基础监控

**章末练习**
- 在小型 Linux 服务器完成部署
- 编写“上线检查清单”

## 第11章：收官与可持续迭代
**预计时间：1 小时**

- 回顾架构与关键决策
- 可扩展方向（评论、媒体库、审计日志等）
- 技术债识别与版本迭代计划
- 教程结课项目建议

**章末练习**
- 给出你的 v2 功能路线图（4~6项）
- 说明每项功能的技术影响范围
